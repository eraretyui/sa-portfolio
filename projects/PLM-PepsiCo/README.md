# PLM-система для ООО «ПепсиКо Холдингс»

**Объект:** ООО «ПепсиКо Холдингс»  
**Период:** 2025–2026  
**Контекст:** Курсовая КНИР + практические работы, НИТУ МИСиС

---

## Проблема
Процесс разработки новой рецептуры велся в разрозненных системах:
Excel-таблицы, локальные БД, ручные согласования. Это приводило к:
- увеличению Time-to-Market
- ошибкам в расчётах и нормативным несоответствиям
- сложностям в управлении версиями данных

## Цель
Спроектировать PLM-систему для автоматизации процесса — от получения
технического задания до формирования итогового пакета документации.

### Акторы и прецеденты

| Актор | Прецеденты |
|-------|------------|
| Директор по инновациям | Просмотреть итоговую рецептуру; Зафиксировать решение; Инициировать разработку; Передать ТЗ |
| Главный технолог | Сформировать концепцию; Утвердить расчёт; Создать файл рецептуры; Скорректировать состав |
| Технолог R&D | Создать рецептуру; Подобрать ингредиенты; Запустить расчёт КБЖУ; Просмотреть историю версий |
| Специалист по нормам | Провести нормативную проверку; Сформировать замечания |

## 4. Модель доступа

### Роли в системе

| Роль | Тип | Описание |
|------|-----|----------|
| Директор по инновациям | Ключевой | Только чтение и фиксация решений, без расчётов |
| Главный технолог | Ключевой | Широкие права: управление документацией, утверждение расчётов |
| Технолог R&D | Основной | Создание и редактирование рецептур, запуск расчётов |
| Специалист по нормам | Основной | Нормативная проверка, формирование замечаний |

### CRUD-матрица

| Объект системы | Директор | Гл. технолог | Технолог R&D | Специалист по нормам |
|----------------|:--------:|:------------:|:------------:|:--------------------:|
| Техническое задание | R | R | R | R |
| Файл рецептуры | R | C R U | C R U | R |
| Расчёт КБЖУ и ФХП | R | R | C R | R |
| Нормативная документация | R | C R U D | R | C R U |
| Управленческое решение | C R | R | — | — |
| История версий | R | R | R | R |

*C — создание, R — чтение, U — изменение, D — удаление*

### Правила доступа
- Директор по инновациям не выполняет расчётов и не создаёт документацию
- Главный технолог может отклонить расчёт и отправить на пересчёт
- Технолог R&D не имеет доступа к управленческим решениям
